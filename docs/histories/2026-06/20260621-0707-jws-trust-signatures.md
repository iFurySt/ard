# JWS Trust Signature Verification

## Request

Continue building the Go/Cobra/Gin/GORM/Postgres ARD registry and toolkit with
spec-aligned trust verification, real tests, and milestone commits.

## Changes

- Added explicit detached compact JWS verification for `trustManifest.signature`.
- Added JSON trust-anchor loading for Ed25519/EdDSA public keys.
- Added `ard verify catalog --jws-trust-anchors`.
- Added `ard verify catalog --require-jws-signatures`.
- Added unit tests that generate real Ed25519 keys, sign trust manifests, verify valid
  signatures, and reject tampered manifests or unknown key IDs.
- Added CLI coverage for signed catalog verification with machine-readable output.
- Updated README, trust, security, architecture, quality, and release documentation.

## Design Intent

The ARD spec defines `trustManifest.signature` as a detached JWS signature, but does not
define key resolution in the local registry. This change keeps default ingestion scoped to
schema and shape validation, then adds an explicit operator-controlled verification gate
for organizations that already maintain trust anchors.

The signed payload is the deterministic JSON generated by `ard` for `trustManifest` with
`signature` removed. This proves the configured Ed25519 key signed the verified metadata,
but it does not prove key ownership, DID/SPIFFE identity, attestation truth, certificate
identity, or runtime safety.
